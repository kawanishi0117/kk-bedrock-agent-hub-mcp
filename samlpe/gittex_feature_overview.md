## **1. サービス概要**

GitTeXは **Git を基盤とした、LaTeX / Markdown ドキュメントの共同編集・バージョン管理サービス**です。

論文、技術資料、研究室プロジェクトなど、文章と数式の厳密な管理が必要な領域で、
単純なクラウドストレージや Google Docs では解決できない、
**「履歴」「差分」「レビュー」「再現性」** の課題を解消します。

---

## **2. 提供価値**

| 課題                          | GitTeXの解決                          |
| ----------------------------- | ------------------------------------- |
| Word や Docs の差分が見にくい | ソースレベルとPDFレベルの差分を可視化 |
| 共同編集の競合                | Gitベースの安全なマージ管理           |
| 論文PDF生成の環境依存         | Docker / Fargateで完全再現            |
| どれが最新版かわからない      | ブランチ運用とタグ付                  |

## **5. 技術構成（予定＋MVP）**

| 要素         | 技術                                           |
| ------------ | ---------------------------------------------- |
| Webアプリ    | React / Next.js / Tailwind                     |
| バックエンド | Lambda / API Gateway                           |
| Git処理      | Lambda + EFS + isomorphic-git                  |
| PDFビルド    | Docker on Fargate / Queue                      |
| 認証         | Cognito or SES認証                             |
| DB           | Aurora PostgreSQL互換                          |
| ストレージ   | EFS（リポジトリ）+ S3（成果物）                |
| AI補助       | Bedrock Claude – summarize / review / refactor |

---

## **6. 差分表示について**

* Git差分を行単位で表示
* 数式変更のみ抽出
* 将来的にPDF比較を実装（差分ハイライト）
